# Data Model: Market Intelligence AI — Hackathon MVP

**Branch**: `001-market-intelligence-mvp` | **Date**: 2026-03-21

No persistent storage — all entities below are in-memory runtime structures
within a single analysis session.

---

## Entities

### AnalysisRequest

The input submitted by the user to trigger the pipeline.

| Field | Type | Constraints |
|-------|------|-------------|
| `keyword` | `str` | Non-empty, stripped of whitespace; 1–200 chars |
| `session_id` | `str` | UUID generated server-side per request |

---

### MarketplaceProduct

A single product retrieved from the marketplace source.

| Field | Type | Constraints |
|-------|------|-------------|
| `title` | `str` | Non-empty |
| `price` | `str` | Formatted price string (e.g., `"$29.99"`) or `"N/A"` |
| `url` | `str` | Absolute product URL |
| `source` | `str` | Always `"amazon"` for MVP |

---

### SourceResult

The raw data returned by a single data source after collection.

| Field | Type | Constraints |
|-------|------|-------------|
| `source` | `str` | `"amazon"` \| `"google_trends"` \| `"reddit"` |
| `status` | `str` | `"success"` \| `"timeout"` \| `"error"` |
| `data` | `dict \| None` | Structured payload; `None` if status ≠ success |
| `error_message` | `str \| None` | Human-readable reason if status ≠ success |

---

### AggregatedData

Normalised view of all available source results, passed to the LLM analysis layer.

| Field | Type | Constraints |
|-------|------|-------------|
| `keyword` | `str` | The original analysis keyword |
| `products` | `list[MarketplaceProduct]` | Empty list if amazon timed out |
| `trend_data` | `dict \| None` | Google Trends payload; `None` if unavailable |
| `reddit_data` | `dict \| None` | Reddit posts payload; `None` if unavailable |
| `available_sources` | `list[str]` | Sources that returned successfully |
| `unavailable_sources` | `list[str]` | Sources that timed out or errored |

**Invariant**: At least one source in `available_sources` for LLM invocation;
otherwise the pipeline emits an error and does not invoke the LLM.

---

### ViabilityScore

The market viability assessment generated by the LLM.

| Field | Type | Constraints |
|-------|------|-------------|
| `score` | `int` | 0–100 inclusive |
| `explanation` | `str` | Human-readable explanation of score drivers |

**Export format**: `Score: {score}/100` on its own line (machine-parsable,
see Constitution Principle III).

---

### TargetPersona

The primary customer segment identified by the LLM.

| Field | Type | Constraints |
|-------|------|-------------|
| `description` | `str` | Full persona narrative |
| `characteristics` | `list[str]` | 3–5 key bullet-point characteristics |

---

### DifferentiationAngles

Actionable differentiation opportunities identified by the LLM.

| Field | Type | Constraints |
|-------|------|-------------|
| `angles` | `list[str]` | 3–5 actionable items |

---

### CompetitiveOverview

The competitive landscape summary generated by the LLM.

| Field | Type | Constraints |
|-------|------|-------------|
| `overview` | `str` | Narrative overview |
| `players` | `list[str]` | Named competitors or categories identified |

---

### AnalysisReport

The complete report assembled at the end of the pipeline, used for export.

| Field | Type | Constraints |
|-------|------|-------------|
| `keyword` | `str` | Original keyword |
| `generated_at` | `datetime` | UTC timestamp |
| `products` | `list[MarketplaceProduct]` | May be empty |
| `viability` | `ViabilityScore \| None` | None if LLM failed |
| `persona` | `TargetPersona \| None` | None if LLM failed |
| `angles` | `DifferentiationAngles \| None` | None if LLM failed |
| `competitive` | `CompetitiveOverview \| None` | None if LLM failed |
| `unavailable_sources` | `list[str]` | Forwarded from AggregatedData |
| `is_partial` | `bool` | True if any source or LLM section failed |

## SSE Event Payload Types

These are the wire types sent over the SSE stream (matches `contracts/sse-events.md`).

### BaseSSEEvent

| Field | Type | Notes |
|-------|------|-------|
| `event_type` | `str` | Matches the SSE `event:` name |
| `status` | `str` | `"streaming"` \| `"complete"` \| `"error"` |

### MarketplaceProductsEvent (event: `marketplace_products`)

| Field | Type | Notes |
|-------|------|-------|
| `status` | `"complete"` | Always complete (not streamed) |
| `products` | `list[MarketplaceProduct]` | |

### LLMSectionStreamEvent (events: `viability_score`, `target_persona`, `differentiation_angles`, `competitive_overview`)

Streaming token:

| Field | Type | Notes |
|-------|------|-------|
| `status` | `"streaming"` | |
| `token` | `str` | Single LLM token |

Complete event:

| Field | Type | Notes |
|-------|------|-------|
| `status` | `"complete"` | |
| `content` | `str` | Full generated content |
| `score` | `int \| None` | Only present on `viability_score` event |

Error event (LLM interrupted — per clarification Q1):

| Field | Type | Notes |
|-------|------|-------|
| `status` | `"error"` | |
| `partial_content` | `str` | Content streamed before interruption |
| `message` | `str` | e.g., `"Generation interrupted"` |

### SourceUnavailableEvent (event: `source_unavailable`)

| Field | Type | Notes |
|-------|------|-------|
| `source` | `str` | `"amazon"` \| `"google_trends"` \| `"reddit"` |
| `message` | `str` | Human-readable reason |

### ExportReadyEvent (event: `export_ready`)

| Field | Type | Notes |
|-------|------|-------|
| `status` | `"complete"` | |

## State Transitions

### Pipeline Lifecycle

```
IDLE
  │ user submits keyword
  ▼
COLLECTING (parallel: amazon, google_trends, reddit — 10s timeout each)
  │ first source resolves
  ▼
EMITTING_PRODUCTS (emit marketplace_products event)
  │ all sources resolved or timed out
  ▼
AGGREGATING
  │ at least 1 source available
  ▼
GENERATING_VIABILITY → GENERATING_PERSONA → GENERATING_ANGLES → GENERATING_COMPETITIVE
  │ all LLM sections complete (or error)
  ▼
COMPLETE (emit export_ready event)
  │ user clicks "New analysis"
  ▼
IDLE
```

### Section Display States (frontend)

```
HIDDEN → LOADING (skeleton) → STREAMING (partial content) → COMPLETE
                                        │
                                        └→ ERROR (partial content + error notice)
```
