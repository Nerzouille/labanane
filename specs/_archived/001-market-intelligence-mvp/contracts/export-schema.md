---

## Markdown Export Schema

The Markdown export is the canonical machine-parsable format. The PDF export
is derived from this schema and is intended for human consumption only.

### File naming

`market-intelligence-{keyword-slug}-{YYYY-MM-DD}.md`

Example: `market-intelligence-eco-bamboo-skincare-2026-03-21.md`

---

### Full schema template

```markdown
# Market Intelligence Report: {keyword}

**Generated**: {YYYY-MM-DD HH:MM} UTC
**Sources**: {comma-separated list of available sources}
{**Partial report**: Sources unavailable: {comma-separated list}  ← omit line if all sources available}

---

## Marketplace Products

{For each product:}
- **{title}** — {price} — [View on Amazon]({url})

{If no products available:}
- *No marketplace data available (source unavailable)*

---

Score: {score}/100

{explanation paragraph}

{If section failed:}
Score: N/A

*Generation interrupted. Partial analysis: {partial_content}*

---

{persona description paragraph}

{If section failed:}
*Generation interrupted. Partial analysis: {partial_content}*

---

## Differentiation Angles

{angles as numbered or bulleted list}

{If section failed:}
*Generation interrupted. Partial analysis: {partial_content}*

---

## Competitive Overview

{overview paragraph}

{If section failed:}

---

*Report generated by Market Intelligence AI*
```

## Machine-Parsable Fields

Consumers (AI agents) MUST use these patterns to extract structured data:

| Field | Pattern | Example |
|-------|---------|---------|
| Viability score (numeric) | `^Score: (\d+)/100$` (regex, line-anchored) | `Score: 74/100` |
| Score unavailable | `^Score: N/A$` | `Score: N/A` |
| Partial report flag | Line starting with `**Partial report**:` | `**Partial report**: Sources unavailable: amazon` |

All section headings are H2 (`##`) with exact titles as above. Heading text
MUST NOT change between exports.
